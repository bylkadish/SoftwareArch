# Лабораторная работа №1
**Тема:** Формулирование требований к программной системе.

**Цель работы:** Научиться анализировать поставленную задачу, формулировать функциональные и нефункциональные требования к проектируемой системе.

## Перечень заинтересованных лиц (стейкхолдеров)

  1. **Конечный пользователь (владелец дома)**

     - Использует систему для управления освещением, дверными замками, камерами и другими устройствами.
     - Заинтересован в надёжности, удобстве и безопасности.
  2. **Производитель системы (компания-заказчик)**

     - Инвестирует в разработку.
     - Отвечает за выпуск продукции на рынок.
     - Заинтересован в масштабируемости, продаже большого количества устройств и сборе данных для аналитики.
  3. **Инженеры по разработке ПО**

     - Проектируют программную архитектуру, клиентские приложения, серверную часть и протоколы взаимодействия.
  4. **Разработчики аппаратных модулей (камеры, замки, термостаты)**

     - Создание физических устройств, которые должны корректно взаимодействовать с программной системой.
  5. **Служба технической поддержки**

      - Обеспечивает сопровождение пользователей, обрабатывая обращения и устраняя неисправности.
  6. **Специалисты по кибербезопасности**

     - Отвечают за защиту пользовательских данных.
     - Предотвращение несанкционированного доступа к устройствам.
  7. **Регуляторы и контролирующие органы разных стран**

     - Следят за соблюдением законов о защите персональных данных и технических стандартов.
